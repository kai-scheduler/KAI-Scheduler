# ci: add explicit contents: write permission to release workflow

## Summary

Adds explicit `contents: write` permission at the job level in the release workflow.

## Motivation

The `Upload As Release Asset` step uses `softprops/action-gh-release@v2` with `GITHUB_TOKEN`, which requires `contents: write` permission to upload assets to a release. Without explicit permissions, the workflow relies on the repository's default token permissions, which may be read-only and cause the step to fail.

## Changes

- Added `permissions: contents: write` to the `build-and-push` job in `.github/workflows/on-release.yaml`, following the principle of least privilege.
